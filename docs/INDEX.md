# docs/ — Discovery Index

This file is the single top-level entry point for the GAIA artifact directories
produced and consumed by the `gaia` plugin's lifecycle workflows. Every GAIA
project materialises the same three directories under `docs/`, and every
workflow (PRD, architecture, sprint planning, dev-story, review gate, test
design, and so on) reads from or writes to one of them. Use the links below to
navigate to the category you need.

> The `gaia-public` repo itself only stores a handful of long-form migration
> docs here (`docs/migration/`, `docs/migration-guide-v2.md`). The three
> artifact directories described below live under each GAIA project's own
> `docs/` tree — they are generated by GAIA workflows in the user's project,
> not vendored into this marketplace repo.

## Artifact Directories

### `planning-artifacts/`

Strategy and up-front design outputs — everything produced before a sprint
starts coding. Typical contents:

- `product-brief.md`, `prd.md`, and market/domain research notes from the
  analyst and PM agents.
- `architecture.md`, ADR entries, readiness-report, and infrastructure design
  from the architect and DevOps agents.
- `ux-design.md` and associated design-system extracts.
- `epics.md` / epic-status outputs from sprint planning.

These files are INDEX_GUIDED by most downstream workflows — dev agents and
review workflows load only the relevant sections, not the whole file.

### `implementation-artifacts/`

Per-story execution outputs — everything produced while a sprint is in flight
or under review. Typical contents:

- `{story_key}-{slug}.md` — the canonical story file for each user story,
  source of truth for sprint state, review-gate rows, acceptance criteria, and
  the Definition of Done.
- `{story_key}-tdd-progress.md` — TDD red/green/refactor phase summaries.
- `{story_key}-review.md`, `{story_key}-security-review.md`,
  `{story_key}-performance-review.md` — per-review reports wired into the
  six-row review gate.
- `sprint-plan-*.md`, `sprint-status.yaml`, `correct-course-*.md`,
  retrospective notes, and other sprint-level operational docs.

`/gaia-dev-story`, `/gaia-run-all-reviews`, and `/gaia-sprint-status` all read
and write files in this directory.

### `test-artifacts/`

Test plans, ATDD specs, QA outputs, and automated-run evidence. Typical
contents:

- `test-plan.md` — the master test plan, including the §11.37 GAIA Native
  Conversion Parity Harness.
- `atdd-{epic}.md` / `atdd-{story_key}.md` — acceptance-test specs for
  high-risk stories, required by the dev-story pre-start gate.
- `{story_key}-qa-tests.md`, `{story_key}-test-automation.md`,
  `{story_key}-test-review.md` — per-story QA workflow outputs.
- `traceability-matrix.md`, `test-gap-*.md`, `ci-setup.md`, mobile and
  performance test plans.
- `test-results/{story_key}-execution.json` — Bridge Layer 3 execution
  evidence (schema FR-194 / FR-197, NFR-034 500KB cap).
